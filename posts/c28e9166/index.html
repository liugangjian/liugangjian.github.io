程序计数器

定义

Program Counter Register 程序计数器（寄存器）

  • 作用
    • 记住下一条jvm指令的执行地址
  • 特点
    • 是线程私有的
    • 不会存在内存溢出

实例

虚拟机栈

定义

Java Virtual Machine Stacks （Java 虚拟机栈）

  • 每个线程都有一个虚拟机栈(线程私有)

  • 每个正在运行的方法都对应着一个活动栈帧

问题辨析

  1. 垃圾回收是否涉及栈内存？

    • 不涉及，栈内存中方法调用完毕后，活动栈帧会自动弹出，栈空间得到释放，故垃圾回收不会也不需要对栈内存进行回收

  2. 栈内存分配越大越好吗？

    • 栈内存大小可以通过虚拟机参数中进行指定

      • Linux(64-bit): 1024KB
      • macos(64-bit): 1024KB
      • Oracle Solaris/x64: 1024KB
      • Windows: The default value depends on virtual memory
    • 栈内存越大，能够同时运行的线程数反而越少
      • 因为一个机器的物理内存大小是一定的，例如: 500M
      • 如果设定每一个线程的栈内存为1M，则同时可以跑500个线程
      • 如果设置栈内存为2M，则同时可以运行250个线程

  3. 方法内的局部变量是否线程安全？

    • 如果方法内局部变量没有逃离方法的作用访问，它是线程安全的

    • 如果是局部变量引用了对象，并逃离方法的作用范围，需要考虑线程安全

栈内存溢出

  • 栈帧过多导致栈内存溢出

    • 本地方法无限递归

    • 第三方方法无限递归

  • 栈帧过大导致栈内存溢出

线程运行诊断

  1. 用top定位哪个进程对cpu的占用过高
  2. ps H -eo pid,tid,%cpu | grep 进程id （用ps命令进一步定位是哪个线程引起的cpu占用过高）
  3. jstack 进程id
  4. 可以根据线程id 找到有问题的线程，进一步定位到问题代码的源码行号

本地方法栈

本地方法栈的功能和特点类似于虚拟机栈，均具有线程隔离的特点以及都能抛出StackOverflowError和OutOfMemoryError异常。

不同的是，本地方法栈服务的对象是JVM执行的native方法，而虚拟机栈服务的是JVM执行的java方法。

JAVA中的native方法需要直接与底层硬件进行交互，这里用java代码是不能实现的，必须使用更加底层的代码如: C++/C, 这部分方法称作本地方法
